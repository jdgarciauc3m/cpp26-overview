\subsection{Pitfalls}

\begin{frame}[t,fragile]{The most contentious feature of C++26}
  \begin{itemize}

    \item \textbad{Contracts} are controversial, with some developers expressing concerns about their complexity, potential for misuse, and impact on safety and correctness.
    
    \mode<presentation>{\vfill}
    \item Some of the main criticisms of contracts include:
    \begin{itemize}

      \mode<presentation>{\vfill}
      \item \textmark{Undefined behavioor}: contracts can lead to undefined behavior if they are violated, which can make it difficult to debug and maintain code that uses them.
      
      \mode<presentation>{\vfill}
      \item \textmark{Incompleteness}: Some key features (e.g. virtual functions, function pointers, \ldots) cannot be used with contracts.
      
      \mode<presentation>{\vfill}
      \item \textmark{Violations of ODR}: Mixed mode builds lead to undefined behavior if the same function is compiled with different contract specifications in different translation units.
      
      \mode<presentation>{\vfill}
      \item \ldots

    \end{itemize}

  \end{itemize}
\end{frame}
